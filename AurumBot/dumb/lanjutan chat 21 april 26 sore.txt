LANGKAH 1: PERSIAPAN DATA (selesai ✅)

  1a. collect_data.py (diupdate):
      - Batch size M1 dinaikkan ke 99,000
      - DATA_LOOKBACK_DAYS = 365
      - Berhasil ambil 99,000 bars M1 (Jan 2026 - Apr 2026, harga 4455-5593)
      - Berhasil ambil 20,000 bars M15 (Jun 2025 - Apr 2026)
      - Disimpan sebagai "Finex data"

  1b. Percobaan baca file cache MT5 (.hc/.hcc):
      - Investigasi file binary MT5 history
      - Format berhasil di-reverse engineer: bar_size=60 bytes, header variable
      - TAPI: file M1.hc (200MB) menggunakan format berbeda yang tidak berhasil dibaca
      - Keputusan: SKIP, gunakan Kaggle saja

  1c. prepare_data.py (BARU, menggantikan convert_kaggle.py):
      - Filter Kaggle dari 2023-01-01 (bukan 2020 atau 2024)
      - Hardcode FINEX_M1_START = "2026-01-08 03:40:00"
      - Hardcode FINEX_M15_START = "2025-06-13 21:15:00"
      - Merge Kaggle + Finex tanpa overlap
      - Hasil M1 : 1,129,272 bars (2023-01-03 to 2026-04-21)
      - Hasil M15: 77,908 bars   (2023-01-03 to 2026-04-21)
      - Ada gap 32 hari (batas Kaggle Oct 2025) → wajar untuk LightGBM

LANGKAH 2: RETRAIN MODEL (selesai ✅)

  train_model.py (diupdate):
  - rolling_slope() di-vectorize pakai numpy sliding_window_view (92x lebih cepat)
  - Progress labeling dengan ETA
  - Validasi data sebelum training

  Hasil training:
  - Data: 1,127,033 candles setelah dropna
  - Price range: 1806.71 - 5593.20
  - Mean ATR14: 1.08 pts (vs 0.80 sebelumnya)
  - Natural WR: BUY 19.78%, SELL 19.55% (tipis di bawah 20%)
  - Train: 901,626 rows (Jan 2023 - Jul 2025)
  - Test : 225,407 rows  (Jul 2025 - Apr 2026) ← mencakup harga terkini
  - BUY model: precision 0.20, recall 0.27, best iteration 115
  - SELL model: precision 0.21, recall 0.44, best iteration 129
  - Top features: m15_slope (1), atr_14 (2), minute (3)

  Profit Factor simulasi:
  - TP=3xATR: BUY PF=1.17, SELL PF=1.09, Avg=1.13
  - TP=4xATR: BUY PF=1.52, SELL PF=0.92, Avg=1.22 ← BEST
  - TP=2xATR: BUY PF=1.05, SELL PF=1.04, Avg=1.05
  - STATUS: PROFITABLE ✅

LANGKAH 3: FIX ATR WARNING (selesai ✅)

  Root cause: rolling(14) default min_periods=14
  → satu NaN di window = hasil NaN
  → candle dengan high==low (doji) di-replace ke NaN → merusak window

  Fix di bot.py:
  1. get_current_atr(): tambah min_periods=max(1, period//2)
  2. get_current_atr(): fallback loop ke bar sebelumnya (max 5 bar)
  3. get_bars() M1: naikkan dari 200 ke 300 candles
  4. Warning message lebih informatif
